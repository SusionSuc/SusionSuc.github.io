ImplLoader编写记录

实战库ImplLoader介绍

当一个Android工程中如果已经使用不同的module来做业务隔离。那我们就可能有这种需求，module1想实例化一个module2的类,一般要怎么解决呢？

  • module1依赖module2
  • 把module2的这个类沉到底层库，然后module1和module2都使用这个底层库。
  • ….等

下面来介绍一个库小库 : ImplLoader。可以很方便解决这个问题。只需这样使用即可:

  1. 使用@Impl标记需要被加载的类

  2. 使用 ImplLoader.getImpl("module2_text_view") 来获取这个类

  3. 初始化ImplLoader

库的代码放在: https://github.com/SusionSuc/ImplLoader

为什么要写这个库 ?

主要是为了练手

在阅读WMRouter和ARouter源码时发现这两个库都用到了自定义注解、自定义gradle插件、Gradle Transfrom API、javapoet和asm库。而我对于这些知识很多我只是了解个大概，或者压根就没听说过。
因此ImplLoader这个库主要是用来熟悉这个知识的。当然这个库的实现思路主要参考WMRouter和ARouter。

实现过程中遇到的一些问题

注解处理器库的创建

整个项目我是建了一个AndroidProject。因为注解库只会在编译的时候用到，因此我单独建了一个Android Library库，用来存放注解处理相关代码。可是在写的时候，发现找不到javax.annotation下注解相关类。后来发现原因是新建的Android Library是不会包含这写库的，需要新建一个Java Library
